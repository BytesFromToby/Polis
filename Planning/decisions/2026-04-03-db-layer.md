# Decision: Database Layer

**Date:** 2026-04-03
**Status:** Accepted

## Context

The API needs to persist users, city templates, active sim runs, per-cycle state snapshots, and narrative events. The architecture decision (SQLite via SQLAlchemy) was settled in `api_spec.md`. This doc covers the implementation of the DB layer itself.

## Decision

Create `scr/db/` as a package with three modules:

- `models.py` — SQLAlchemy ORM table definitions
- `session.py` — engine and `SessionLocal` factory; `get_db()` dependency for FastAPI
- `seed.py` — seeds official city templates from `data/` into the DB at server startup

### Tables

**`users`**
- `user_id` UUID PK, `username` unique, `email` nullable, `password_hash`, `description`, `is_gm` bool default true

**`cities`**
- `city_id` UUID PK, `city_name`, `author`, `description`, `setting` default "DnD", `details`, `is_official` bool
- `published` bool default false
- `domains_json`, `factions_json`, `units_json`, `world_state_json` — TEXT blobs (serialized via `serializer.py`)

**`sim_runs`**
- `run_id` UUID PK, `user_id` FK, `city_id` FK
- `current_cycle` int default 0, `status` string ("setup" | "running" | "paused" | "complete")
- `setting`, `description`, `details` — carried from the city template at run creation
- `created_at`, `updated_at` datetime

**`cycle_snapshots`**
- `snapshot_id` UUID PK, `run_id` FK, `cycle_number` int
- `state_json` TEXT — full output of `serialize_state()`
- `created_at` datetime

**`narrative_log`**
- `log_id` UUID PK, `run_id` FK, `cycle_number` int
- `events_json` TEXT — list of `serialize_cycle_event()` dicts
- `created_at` datetime

### Seeding

`seed.py` checks whether official city templates exist in the DB. If not, it loads them from `data/` using `loaders.py`, serializes the state with `serializer.py`, and inserts them as `is_official=True` city rows.

The Rivers Point data directory (`data/`) is the first seeded template.

## Rationale

Keeping the DB layer as its own package separates persistence concerns from the API routing layer. `session.py` provides a single place to configure the database URL and a `get_db()` generator that FastAPI injects as a dependency. `seed.py` runs at server startup so templates are always present without manual migration steps.

## Consequences

- The API can import `get_db` from `db.session` and use it as a FastAPI dependency
- City templates are queryable from the DB, not read from files at request time
- State restoration on server restart is done by querying the latest `cycle_snapshot` for a `sim_run`
- `data/` JSON files remain as the authoring source for official cities; they are not read at runtime

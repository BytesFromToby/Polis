# Withhold — the supply-interruption primitive

**Date:** 2026-06-16
**Spec impact:** `actions_spec.md`, `public-needs_spec.md`, `faction-behavior_spec.md`,
`events_spec.md`, `cycle-runner_spec.md`; reference `architecture.md`, `data-models.md`.
**Proposal:** `proposals/resource-chains.md` → "Withhold — the supply-interruption primitive".

## What

A faction can refuse to deliver its trade, zeroing its chain contribution for the cycle and
squeezing the city. Toil's evil twin: Toil is the cycle-only flag at ×1.5, Withhold is the same
flag at **×0**. The same flag is also set by an event for its duration (a storm closing the sea).

## Decisions

**1. Pure opportunity cost — no extra bite on the withholder.** Like Toil, the only cost is the
forgone action; no rank/health/reputation penalty on the striker. The pain is entirely
downstream (shortage → Hungry Public → support drains from the Mayor). Symmetric with Toil and
the least new machinery. (User call: "Pure opportunity cost.")

**2. One cycle-only flag, not a countdown.** `Faction.withholding: bool`, the exact twin of
`toiling` — set in the action loop (or asserted by an active event), consumed by the needs step,
reset at cycle end. Honors the proposal's "same flag as Toil" framing; no inventory, no
"who-set-it" bookkeeping. Checked before Toil in the chain — Withhold wins if both are somehow set.

**3. Event-forced withhold is in scope — and it forced one cycle-spine change.** (User call:
"Also include event-forced withhold now.") A single cycle-only flag can only serve event-forced
withhold if active-event **effect application** runs *before* the needs step — otherwise the
event sets the flag at the old Step 8 (after needs already ran) and the end-of-cycle reset wipes
it before it is ever read. So `process_active_events` moves to item **5a**, immediately before
the needs step; new-event **rolling** stays after needs (its band gates must see this cycle's
freshly-drifted bands). An active storm re-asserts the flag each cycle for its `duration`; the
city recovers the cycle after the event resolves.
- *Why this is low-risk:* the needs step reads only faction `level` + the toil/withhold flags. It
  does **not** read any field active events mutate (health/rating/drift/chaos), so moving those
  applications earlier changes nothing for needs — only the new `withhold` field matters, which is
  the point. The full suite + inspector guard the blast radius.
- *Alternative rejected:* a second persistent `withheld_cycles` countdown field (no reorder). It
  avoids touching the spine but splits the concept across two fields and introduces an
  action-vs-event off-by-one on the shared field. The reorder keeps one flag and one clear story;
  preferred for legibility over spine-stability here because the move is provably needs-neutral.

**4. Structurally rare, not tuned-rare — base weight 0.** Nothing reaches for Withhold by
default. It earns weight only from anger: the Mayor's standing **with that faction**
(`mayor.get_reputation(faction.id)`) being low (≤ `WITHHOLD_ANGER_THRESHOLD`, default −20), scaling
with how far below. This is the already-modeled per-faction signal — no new "angry at mayor"
trait needed. Requires plumbing `mayor` into `decide_action` (a small, justified pass); absent a
mayor (headless), Withhold stays at 0. Every strike is legible as a consequence of alienating
that faction.

**5. Withhold is not a Mayor-proposable audience term.** The deal that *ends* a strike is the
faction committing to a non-withhold action (Toil/Grow/abstain) — the existing audience terms
already settle it. The generic `committed_action` machinery accepts "Withhold" (so a deal/event
*can* force it), but `audience_spec.md` is unchanged: the Mayor never offers "go on strike."

## Property preserved

Source redundancy is untouched: one withholder (or one storm) drives the Public Hungry, never
Starving from full health — it takes a second lost Food source to Starve (the fishery-slice
redundancy property). The balance fixture is the inverted *Toil matters*: one angry level-4
producer that withholds yields a strictly lower minimum `fed` than acting normally, but not
collapse.

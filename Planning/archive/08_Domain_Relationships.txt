================================================================================
CITY SIMULATION SYSTEM
08 — Domain Relationships
v0.3  —  COMPLETE
================================================================================

Domain relationships define how each domain's NPC factions and units
naturally orient toward other domains. These are built-in city properties
— not individual faction decisions. They bias NPC action selection and
target selection automatically each cycle.

The self-referential entry (domain pointing to itself) defines the default
relationship between factions within that domain. Specific faction-to-faction
relationships override this default. See 01_Data_Models for faction.relationships.


================================================================================
RELATIONSHIP TRAITS
================================================================================

trait     NPC behavior effect
---------------------------------------------------------------------------
Friend    +weight toward Assist, Support, Recruit targeting this domain
Foe       +weight toward Attack, Block, Steal targeting this domain
Hide      +weight toward Obscure when this domain is active
          -weight toward actions drawing this domain's attention
Client    +weight toward Spy, Assist, Recruit targeting this domain
          Seeking something specific from this domain
Neutral   No modification to action weights


================================================================================
RESOLUTION ORDER FOR SAME-DOMAIN TARGETING
================================================================================

When faction A targets faction B in the same domain:
  1. Check faction_A.relationships for faction_B — use if found
  2. Fall back to domain.relationships[same_domain] — intra-domain default


================================================================================
HOW RELATIONSHIPS FEED INTO NPC BEHAVIOR
================================================================================

When an NPC unit or faction selects a target this cycle:

1. Start with trait-based action weights (see 04_Traits)
2. Apply focus bias if focus score > inertia
3. Apply domain relationship modifiers:
     For each domain in relationships array:
       if trait == Friend:  boost probability of helpful actions toward that domain
       if trait == Foe:     boost probability of hostile actions toward that domain
       if trait == Hide:    suppress visible actions, boost Obscure
       if trait == Client:  boost Spy and approach actions toward that domain
4. Apply 3-5% wildcard override

================================================================================
ACTION SELECTION PRIORITY
================================================================================

each cycle per NPC unit:
  1. focus_1.score > inertia  →  directed action toward focus_1.target
  2. focus_2.score > inertia  →  directed action toward focus_2.target
  3. else: weighted random from trait action pool
  4. apply 3–5% wildcard override regardless of weights

directed action type selection:
  still filtered through trait weights toward that target
  domain relationships bias target selection (see 08_Domain_Relationships)
  prerequisite gates enforced (Spy before Attack, etc.)
  if gate not met: take gate prerequisite action this cycle instead


================================================================================
TENSION MODIFIER
================================================================================

tension = focus_1.score + focus_2.score

tension 0–4:    weights near baseline
tension 5–7:    primary trait weights × 1.5, others compressed
tension 8–10:   primary trait weights × 2.0, near-exclusive


================================================================================
DOMAIN RELATIONSHIP TARGET BIAS
================================================================================

When selecting a target for an action, domain relationships modify
the probability of selecting units/factions in each domain:

  Friend domains:   +weight for Assist, Support, Recruit targets
  Foe domains:      +weight for Attack, Block, Steal targets
  Hide domains:     avoid as targets for visible actions
                    +weight for Obscure when Hide domain is active
  Client domains:   +weight for Spy, Assist, Recruit targets

See 08_Domain_Relationships for full arrays per domain.


================================================================================
CYCLE RESOLUTION ORDER
================================================================================

step    action
---------------------------------------------------------------------------
1       Units declare (NPC: auto-selected via behavior engine)
2       Factions declare
3       GM declares Force Actions, sets World Chaos per domain
4       GM / Force Actions resolve
5       Faction actions resolve (using faction_effective_power from supporters)
6       Unit actions resolve
7       Apply end-of-cycle changes:
          Entrench updates
          Focus decay and swap check
          Leadership Need tick
          Faction inertia changes
          Health decay (natural + modifiers)
          Health retirement check
          Join desire recalculate
          Reset active_supporters to empty for all factions
8       World Chaos check — fire extraordinary action if triggered
9       Cascade events process
10      Faction capacity check — kick or recruit if triggered
11      Unit Generator — 1 new unaffiliated unit created
12      Emergence check — 1% per faction for level 1 → level 2
13      Cycle counter +1

================================================================================
ACTION SELECTION MODEL
================================================================================

Each cycle an actor selects actions by allocating action points
proportionally based on action weights.

BASE WEIGHTS (all actions start here):
  Protect         10.10
  Care            10.09
  Support Faction 10.08
  Obscure         10.07
  Grow            10.06
  Join            10.055
  Leave           10.054
  Block           10.05
  Recruit         10.045
  Kick            10.044
  Evolve          10.043
  Expose          10.03
  Steal           10.02
  Harm            10.01
  Seek Leadership 10.00

Decimal tiebreakers determine priority when weights are equal.
Trait modifiers add to base weights.
Multiple traits stack — all modifiers apply.

POINT ALLOCATION:
  Calculate each action's share of total weight pool
  Allocate action points proportionally
  Floor each result
  Remaining points go to highest weighted action
  Actions with 0 allocated points do not fire

CARE ACTION — HEALTH-BASED WEIGHT REDUCTION:
  If unit health > 75:
    Care weight -= (health - 75)
  floor: Care weight cannot drop below 0.01
  examples:
    health 80:  Care weight -5
    health 90:  Care weight -15
    health 75:  no change

PROTECT ACTION — ENTRENCH-BASED WEIGHT REDUCTION:
  If unit primary domain entrench > 0.85:
    Protect weight -= 1 per 0.01 over 0.85
  floor: Protect weight cannot drop below 0.01
  examples:
    entrench 0.90:  Protect weight -5
    entrench 0.95:  Protect weight -10
    entrench 0.85:  no change

FOCUS OVERRIDE:
  If focus_1.score > inertia — directed action toward focus_1.target
  If focus_2.score > inertia — directed action toward focus_2.target
  Trait weights still determine action type toward that target
  Focus overrides target selection not action type

GATE CHECK:
  If selected action gate not met:
  points for that action redirect to gate prerequisite

================================================================================
CARE ACTION — HEALTH-BASED WEIGHT REDUCTION
================================================================================

updated:    If unit health > 75, Care weight reduced by (health − 75)

examples:
  health 80:   Care weight −5
  health 90:   Care weight −15
  health 95:   Care weight −20
  health 75:   no change
  health 50:   no change

floor:      Care weight cannot drop below 0.01

reason:     Healthy units were wasting action points on Care they didn't
            need. This makes Care self-regulating — units only prioritize
            healing when they actually need it.

affects:    npc.py — select_unit_actions()

================================================================================
PROTECT ACTION — ENTRENCH-BASED WEIGHT REDUCTION
================================================================================

updated:    If unit primary domain entrench > 0.85, Protect weight
            reduced by 1 for every 0.01 over 0.85

examples:
  entrench 0.90:   Protect weight −5
  entrench 0.95:   Protect weight −10
  entrench 1.00:   Protect weight −15
  entrench 0.85:   no change
  entrench 0.50:   no change

floor:      Protect weight cannot drop below 0.01

reason:     Well-entrenched units were spending all their points on
            Protect despite already being fortified. This frees up
            action points for more interesting actions once a unit
            is well-established.

affects:    npc.py — select_unit_actions()

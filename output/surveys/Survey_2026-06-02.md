## Recommendations
1. **[HIGH] Retrofit `Done when:` contracts onto the 11 untagged engine specs.** cycle-runner,
   actions, faction-behavior, personality-system, mayor, treasury, projects, special-factions,
   events, llm-system, llm-profiles each describe behavior but carry **no tagged acceptance
   criteria**. The engine *is* heavily tested (276 tests), but those tests aren't traceable to
   spec criteria, so neither `surveyor` nor `inspector` can prove the specs are met. Run
   **architect** per spec to add `[automated]`/`[human-required]` Done-when items, mapping each to
   an existing test where one already exists. This is the single highest-leverage maintenance
   action for the project. Full-Path (architect) — do not auto-apply. Effort: large (spread it
   across the 11 specs).
2. **[MEDIUM] Run a full `inspector` proof-baseline.** Tests pass, but no dynamic Done-when proof
   was run this pass (the Vue UI + 15 specs + playwright is its own focused effort). Schedule it
   after Rec 1 so inspector has tagged criteria to verify. Full-Path.
3. **[LOW] Update `tools/audit.py`** expected-files list to drop the removed `unit.py`/`membership.py`
   actions (intended removal, 2026-05-25). Stops a misleading warning. Quick-ish (tool edit).
4. **[LOW] Resolve the `budget_allocation` literal in `llm-system_spec.md`** — reword the cross-ref
   note or relax the audience criterion; add the repo-wide test if the strict form is intended.
   Full-Path (touches a spec) — reconcile via architect.

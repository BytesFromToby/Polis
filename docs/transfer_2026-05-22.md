## Known issues / things to watch

- **LLM term format**: The model (qwen3:8b) sometimes returns deal terms as plain strings (`"public_endorsement"`) rather than structured dicts. The parser now maps common strings via `_STRING_TERM_MAP` in `response_parser.py`, but unusual LLM-invented terms will still be silently dropped. Expand the map if new patterns appear.
- **Faction terms in deals**: `faction_terms` from the LLM (e.g. `"refrain from harm against X for 5 cycles"`) are freeform strings that don't map to engine `DealTerm` types, so they are skipped by `_parse_terms`. The deal still gets created; it just has no engine-enforced faction obligations. This is a known gap — faction deal enforcement is not yet implemented.
- **`audience_state` not persisted**: The in-progress audience state lives only in `SimSession` (in-memory). A server restart mid-audience will lose it. This is acceptable for now.
- **5-cycle cooldown on MeetWithFaction**: Set at conclude, not at begin. If the server crashes mid-audience the AP is spent but no cooldown is set.

## Architecture quick-reference

```
/scr
  api/
    routes/mayor.py       ← audience endpoints (/begin /reply /conclude) + all mayor actions
    routes/sim.py         ← start/step/status + PATCH /sim/llm-profile
    sessions.py           ← SimSession (holds audience_state, llm_profile_id)
    schemas.py            ← AudienceBegin/Reply/Conclude request+response schemas

  engine/
    llm/
      audiences.py        ← begin_audience_step / reply_audience_step / conclude_audience_step
      client.py           ← LLMClient (openai_compat + anthropic), LLMConfig
      response_parser.py  ← <deal> block parser, _try_parse_json, _STRING_TERM_MAP
      prompt_builder.py   ← system prompt for faction persona
    mayor/
      actions.py          ← all 13 mayor action handlers, ACTION_COSTS, MEET_COOLDOWN
    models.py             ← Mayor, Faction, Deal, DealTerm, etc.

/frontend/src
  components/
    AudienceModal.vue     ← 5-step audience chat UI
    MayorActionsModal.vue ← mayor actions panel; opens AudienceModal for MeetWithFaction
    LLMSettings.vue       ← profile CRUD + active-city profile selector
  api.js                  ← mayor.audienceBegin/Reply/Conclude, sim.setLlmProfile
```

## Suggested next steps (player mentioned / logical follow-ons)

1. **AI integration into cycle events** — LLM profiles are set up and wired to the session, but the cycle runner still uses stub narratives. The next big feature is using the LLM to generate faction event narratives during `run_cycle`.
2. **Faction deal enforcement** — `faction_terms` from audience deals have no engine effect. Implementing `committed_action` / `committed_abstain` term enforcement in `cycle/runner.py` would make deals meaningful.
3. **Audience prompt tuning** — qwen3:8b sometimes puts `<deal>` blocks in steps 1 and 3 (supposed to be prose only). The system prompt in `prompt_builder.py` could be tightened to instruct the model to only include a `<deal>` block in the final turn.
4. **Dashboard/GameView polish** — cycle events log, faction detail view, deal tracker in the Mayor panel.
